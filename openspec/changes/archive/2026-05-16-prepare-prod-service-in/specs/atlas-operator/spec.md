## ADDED Requirements

### Requirement: Backend repository SHALL host an Atlas migration prod overlay

The `liverty-music/backend` repository SHALL contain a `k8s/atlas/overlays/prod/` directory whose `kustomization.yaml` renders the prod-flavored Atlas `AtlasMigration` resources for the `liverty-music-prod` Cloud SQL instance. The directory's existence is a precondition for the corresponding ArgoCD Application (`backend-migrations` in `cloud-provisioning/k8s/argocd-apps/prod/`) to sync: without the path, ArgoCD reports `Status=Unknown (ComparisonError)` because Kustomize manifest generation fails.

The prod overlay SHALL diverge from the existing dev overlay in:

1. Target Cloud SQL instance — the `AtlasMigration.spec.cloud.url` (or equivalent connection string) SHALL reference the prod PSC DNS name (`298474959c18.25pf3r4b6sfkn.asia-northeast2.sql.goog`) and database `liverty-music`.
2. IAM database user — the Atlas connection SHALL authenticate as `backend-app@liverty-music-prod.iam` (Atlas Operator currently uses the backend-app GSA for migration apply, mirroring dev's pattern).
3. Migration source ConfigMap — generated by `configMapGenerator` from the same `migrations/` directory as dev (no fork; both envs apply the same forward migrations).

#### Scenario: Prod overlay directory exists

- **WHEN** listing `liverty-music/backend:k8s/atlas/overlays/`
- **THEN** the directory SHALL contain both `dev/` and `prod/` subdirectories
- **AND** each SHALL contain a `kustomization.yaml`

#### Scenario: Prod overlay renders cleanly

- **WHEN** running `kubectl kustomize k8s/atlas/overlays/prod` from the `liverty-music/backend` repo root
- **THEN** the command SHALL exit with code 0
- **AND** SHALL emit at least one `AtlasMigration` resource targeting the prod Cloud SQL connection

#### Scenario: ArgoCD prod backend-migrations Application reaches Synced/Healthy

- **WHEN** the `backend-migrations` ArgoCD Application in the prod cluster polls its source (`liverty-music/backend:k8s/atlas/overlays/prod` on `main`)
- **THEN** the Application's `status.sync.status` SHALL transition from `Unknown` to `Synced`
- **AND** `status.health.status` SHALL remain `Healthy`
- **AND** no `ComparisonError` condition SHALL be present
