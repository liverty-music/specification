## Follow-up (out of scope for this change, deferred to future work)

These were identified during §3.5 / §4.x live ops verification but do not belong in `playwright-password-test-user`:

- **cloud-provisioning**: Add a `ZitadelHumanUserPasswordPermanent` Pulumi Dynamic Resource that calls Zitadel Management API's `SetPassword(noChangeRequired=true)` after the `zitadel.HumanUser` is created. The `@pulumiverse/zitadel.HumanUser` resource does not expose a `changeRequired: false` knob today, so the capture script has to tolerate the "change password on first sign-in" prompt as a workaround. Replacing that workaround with the Dynamic Resource would make the user fully Pulumi-managed without first-sign-in friction.
